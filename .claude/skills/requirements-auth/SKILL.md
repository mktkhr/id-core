# 要求認可突合スキル

要求文書の権限記述を認可マスター正本と突合し、差分があれば停止する。

## 正本

`docs/context/authorization/matrix.md` (唯一の認可マトリクス実体)

詳細ルールは `.rulesync/rules/authorization-matrix.md` に従う。

## 手順

1. 要求文書 (`docs/requirements/{N}/index.md`) から権限記述を抽出する
2. マスターの該当セル (機能 × ロール) を抽出する
3. 一致判定する
4. 差分がある場合は停止してユーザー判断を求める
5. 差分ゼロなら要求文書に「突合済」を記録する

## 停止テンプレ

```
【認可マトリクス差分を検出しました】
対象: {機能名}
ロール: {role}
マスター: {空欄/o/▲}
要求文書: {空欄/o/▲}
選択肢:
  A) マスターに合わせる
  B) マスター変更を提案する (要承認)
     → 承認された場合は /spec-resolve の「マスター変更の正規フロー」で反映
       (ADR 起票 → 正本 docs/context/authorization/matrix.md 更新 → 要求文書の認可突合を再実行)
  C) 保留 (未決事項へ)
```

## 禁止

- マスターの値を推測で変更しない
- 差分未解消のまま次フェーズへ進めない
