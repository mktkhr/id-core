# 差分追跡スキル

設計書 (`docs/specs/` 配下) の「変更履歴」セクションに、直近の変更を追記する。

## 前提として読み込む context

**必須** (必ず読む):
- 対象の `docs/specs/{N}/index.md` (既存「変更履歴」セクションのみで足りる場合は部分読み可)

**条件付き**: なし

**読み込まない**: context/* 配下は不要。差分の追記のみが目的。

詳細は `docs/context/README.md` の対応表を参照。

## 入力

- `$ARGUMENTS` から対象の設計書パスを特定する
- 引数なし: 会話コンテキストから対象設計書を推定する

## 実行手順

### 1. 現在の変更履歴を読み取る

対象設計書の「## 変更履歴」セクションを読み取り、既に記録済みの変更を把握する。

### 2. 直近の変更を特定する

会話コンテキストから、前回の差分追記以降に行った変更を特定する。対象:

- DB 設計の変更 (ER 図、DDL、制約、型、カラム追加 / 削除)
- API 設計の変更 (エンドポイント追加 / 変更、認可、エラーコード)
- 認可マトリクスのコピー差分
- 論点解決による業務ルールの変更

### 3. 変更履歴に追記

テーブル形式 (日付 / 変更内容) に従って追記する。

- 既存の行は変更しない (追記のみ)
- 重複する行がないか確認する
- カテゴリ (DB 設計 / API 設計 / 認可) ごとに整理する

### 4. ユーザーに報告

追記した変更の件数と概要を報告する。

## spec-resolve との二重追記防止

`spec-resolve` は決定内容を「変更履歴」に追記する責務を持つ。
`spec-track` では **spec-resolve で未追記の変更のみ** 反映する。同一決定の二重追記を避ける。
