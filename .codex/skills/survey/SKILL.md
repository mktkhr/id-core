# Survey

## 調査

- ユーザーから依頼された内容の調査をしてください
- 調査結果は md ファイルで、`.ai-out/` 配下に保存してください (なければ作成)
- 英語で思考し、日本語で結果を記載してください
- md ファイル名は `survey-{現在日時:yyyyMMddHHmmss}_{調査内容を一言で表現するもの}.md`
- 以下のフォーマットで調査結果をまとめてください

## 調査対象

このリポジトリ内で完結する調査が中心:

- ドキュメント (`docs/` 配下: 設計書・要求・ADR・context)
- id-core 本体のコード (`core/`)
- サンプルアプリのコード (`examples/go-react/`, `examples/kotlin-nextjs/`)
- 開発環境設定 (`docker/`, `Makefile`)
- スキル・ルール (`.rulesync/`)

参考資料 (`アーカイブ/`) はユーザーが明示的に求めた場合のみ参照する。

## 保存フォーマット

```md
## 調査内容

- 何について調査したのかを構造化して書いてください

## 結果

- 調査した結果を書いてください
- 必須項目
  - 対象のファイルパス
  - 対象の設計書セクション / テーブル名 / API 名
  - 対象ファイルや設計要素の関連性・依存関係
- 任意項目 (あれば記載する)
  - 変更すべき箇所
    - 変更によるメリット、デメリット
  - 残タスク
  - その他、ユーザーから指示された内容

## バージョン

- 調査時点でのコミットハッシュ
- 調査対象のブランチ
- 調査実施日
```

## 注意事項

- リポジトリ外への参照は行わない (モノレポ前提)
- WebSearch / WebFetch が必要な場合はユーザーに確認する
- 大量のソースを読み込む場合は Explore サブエージェントの使用を検討
